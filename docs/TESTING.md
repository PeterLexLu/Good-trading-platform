# SGC-丰盛有鱼测试文档

## 测试环境

- 预览服务：`python3 -m http.server 4173`
- 访问地址：`http://localhost:4173`
- 浏览器：Chrome / Safari / Codex in-app browser 均可
- 数据来源：浏览器 `localStorage` 模拟后端

## 快速检查

### 1. 语法检查

在项目目录执行：

```bash
for f in *.js; do node --check "$f" || exit 1; done
```

预期结果：没有报错。

### 2. 页面可访问检查

逐个打开以下页面，确认无空白页、无明显布局错位：

- `http://localhost:4173/index.html`
- `http://localhost:4173/idle.html`
- `http://localhost:4173/wish.html`
- `http://localhost:4173/messages.html`
- `http://localhost:4173/profile.html`
- `http://localhost:4173/detail.html?id=idle-1`
- `http://localhost:4173/publisher.html?id=wechat-user`
- `http://localhost:4173/chat.html?item=idle-1`

## 核心流程测试

### 首页

步骤：

1. 打开 `index.html`
2. 查看海报区域
3. 等待 3-4 秒

预期：

- 页面只展示顶部导航和海报轮播
- 海报能自动切换
- 导航文字居中显示

### 闲置浏览

步骤：

1. 打开 `idle.html`
2. 查看闲置卡片
3. 点击左侧分类
4. 在搜索框输入关键词，例如“血压”
5. 切换城市筛选

预期：

- 默认显示闲置列表
- 分类切换后列表刷新
- 搜索后只显示匹配内容
- 城市筛选能过滤列表
- 无结果时显示空状态

### 心愿池浏览

步骤：

1. 打开 `wish.html`
2. 查看心愿卡片
3. 使用分类、搜索、城市筛选

预期：

- 默认显示心愿列表
- 筛选逻辑和闲置页一致

### 发布闲置

步骤：

1. 打开 `idle.html`
2. 点击“发布闲置”
3. 上传图片或不上传图片
4. 填写物品名称、物品信息、城市、分类
5. 点击“发布”

预期：

- 发布弹窗能打开和关闭
- 图片上传后能预览
- 发布后弹窗关闭
- 新发布内容出现在闲置列表
- 刷新页面后新内容仍存在

### 发布心愿

步骤：

1. 打开 `wish.html`
2. 点击“发布心愿”
3. 填写心愿名称、心愿信息、城市、分类
4. 点击“发布”

预期：

- 新心愿出现在心愿池列表
- 刷新页面后仍存在

### 详情页

步骤：

1. 打开 `idle.html`
2. 点击任意卡片

预期：

- 进入 `detail.html?id=...`
- 展示图片、标题、描述、分类、城市、发布者、发布时间
- 详情页内容只读，没有编辑入口

### 发布者主页

步骤：

1. 打开任意详情页
2. 点击“发布者”

预期：

- 进入 `publisher.html?id=...`
- 展示发布者头像、名称、简介、城市、统计
- 下方显示 TA 发布的闲置和心愿

### 聊一聊

步骤：

1. 打开任意详情页
2. 点击“聊一聊”
3. 在聊天页输入消息并发送
4. 打开 `messages.html`

预期：

- 进入 `chat.html?item=...`
- 自动创建或复用已有会话
- 消息发送后出现在聊天记录中
- 消息列表显示该会话
- 点击消息列表会进入对应聊天页

### 我的主页

步骤：

1. 打开 `profile.html`
2. 点击“微信扫码登录”
3. 点击“模拟扫码成功”
4. 查看我的主页
5. 点击“退出登录”

预期：

- 未登录时显示“未登录”
- 模拟扫码后显示“微信用户”
- 我的主页样式和发布者主页一致
- 统计区域更新
- 我发布的内容展示在下方
- 退出后恢复未登录状态

### 演示数据管理

步骤：

1. 打开 `profile.html`
2. 点击“导出演示数据”
3. 点击“清空演示数据”

预期：

- 导出 JSON 文件
- 清空后恢复示例数据和未登录状态

## 回归测试清单

每次改代码后至少检查：

- 顶部导航能跳转到所有页面
- 闲置和心愿列表能显示
- 详情页能打开
- 聊天页能打开
- 发布者主页能打开
- 我的主页和发布者主页样式一致
- 所有按钮文字上下左右居中
- JS 语法检查通过

## 已知测试限制

Codex in-app browser 的自动化环境可能无法使用脚本填充输入框，会出现虚拟剪贴板限制。遇到这种情况时，输入框相关流程需要手工测试。

这不代表页面输入功能有问题；普通浏览器中可以直接输入。
